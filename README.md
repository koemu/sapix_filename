# sapix-filename

SAPIX のスキャンPDFに対して、表紙からファイル名を推定し、必要に応じてリネーム（または `mv` コマンド形式で出力）する CLI ツールです。

また、PDF のフッターのページ番号を検証し、スキャンの欠落や並び不正を検出します。

## インストール

このリポジトリ直下で以下を実行します。

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -e .
```

## 使い方（コマンドライン）

### 基本

```bash
sapix-filename /path/to/orig.pdf
```

デフォルトでは **実ファイルは変更せず**、標準出力に `mv` コマンド形式で出力します。

### `mv` 形式での出力

```bash
sapix-filename /path/to/orig.pdf
```

出力例（フルパス）:

```bash
mv "/abs/path/orig.pdf" "/abs/path/H350-01_orig.pdf"
```

### 推定ファイル名だけを出力

```bash
sapix-filename --name-only /path/to/orig.pdf
```

出力例:

```text
H350-01_orig.pdf
```

### リネームを実行する

```bash
sapix-filename --apply /path/to/orig.pdf
```

成功時は新しいファイル名を標準出力に出します。

## 推定されるファイル名の形式

推定できた場合、基本的に以下の形式になります。

```text
<推定stem>_<元ファイル名stem>.pdf
```

例:

```text
H350-01_orig.pdf
```

表紙のフォーマットは優先順位の高い順に試行します。最初に一致したフォーマットで処理を終えます。

### 1つ目のフォーマット（GS特訓）

表紙テキストに `GS特訓` がある場合に適用されます。続くトークンと教科をもとにファイル名を構成します。

| トークン例 | 教科 | 出力ファイル名 |
|---|---|---|
| `GTK-01①` | 算数（強制） | `算数GTK-01①_orig.pdf` |
| `GS-01` | 表紙の教科（例: 社会） | `社会GS-01_orig.pdf` |
| `01` | 表紙の教科（例: 理科） | `理科01_orig.pdf` |

- `GTK-` で始まるトークンは教科名が表紙にない場合でも強制的に `算数` を付与します。
- それ以外のトークンは、表紙にある `国語 / 算数 / 理科 / 社会` の記載を先頭に付与します。
- このフォーマットでは `_Answer_` / `_Question_` / `_Exam_` の付与は行いません。

### 2つ目のフォーマット（算数基礎力定着テスト）

表紙テキストに `算数基礎力定着テスト` があり、続く `06①` のようなトークンが読み取れる場合:

```text
算数基礎力定着テスト06①_orig.pdf
```

このフォーマットでは `_Answer_` / `_Question_` / `_Exam_` の付与は行いません。

### 3つ目のフォーマット（四角枠のテキスト番号）

表紙の四角枠の「テキスト番号」を推定します（例: `H350-01`, `WS-01`, `62A-01`, `640-01`, `61-01`）。

```text
H350-01_orig.pdf
62A-01_orig.pdf
```

#### `WS-` の場合の教科名付与

テキスト番号が `WS-` から始まる場合、表紙から教科（`国語/算数/理科/社会`）を抽出して先頭に付与します。

```text
国語WS-01_orig.pdf
```

#### 内容に応じたタグ付与（3つ目フォーマットのみ）

表紙（または文書内）の文言に応じて、以下のタグが付与されます。

| 条件 | タグ | 出力例 |
|---|---|---|
| `解答と解説` を含む | `_Answer_` | `H350-01_Answer_orig.pdf` |
| `国語` かつ `問題・解答用紙` を含む | `_Question_` | `H350-01_Question_orig.pdf` |
| `入試演習問題` を含む | `_Exam_` | `61-01_Exam_orig.pdf` |

`問題・解答用紙` は、PDF の文字抽出時に空白や改行が混ざった場合も同じ文言として扱います。

## AI（生成AI / Vision）の利用

スキャンPDFでは `get_text()` で文字が取れないことがあるため、必要に応じて OpenAI の Vision を使って抽出精度を上げます。

### API キー設定

```bash
export OPENAI_API_KEY="..."
```

環境変数名は `--api-key-env` で変更できます。

### AI を無効化

```bash
sapix-filename --no-ai /path/to/orig.pdf
```

### 使用モデルを変更

```bash
sapix-filename --ai-model gpt-5.4-mini /path/to/orig.pdf
```

## ページ番号の精査

実行時に、PDFのフッターにあるページ番号を検証します（2ページ以下は対象外）。

- ページ番号が重複している
- ページ番号が連続していない
- （全ページに番号がある場合に限り）最終ページ番号と実際のページ数が一致しない

上記に該当すると標準エラー出力へメッセージを出し、終了コード `1` で終了します。

## 付録

### 対応ファイルの形式

このツールが対象とする PDF ファイルの特徴は以下の通りです。

#### ファイル形式

- **形式**: PDF のみ
- **内容**: スキャナーで取り込んだ画像ベースの PDF（テキストレイヤーなし、または不完全）

#### 表紙（1ページ目）のフォーマット

以下の3種類のフォーマットに対応しています。優先順位の高い順に検出を試みます。

| 優先順位 | フォーマット名 | 検出キーワード | 出力例 |
|---|---|---|---|
| 1 | GS特訓 | `GS特訓` | `社会GS-01_orig.pdf`, `算数GTK-01①_orig.pdf` |
| 2 | 算数基礎力定着テスト | `算数基礎力定着テスト` | `算数基礎力定着テスト06①_orig.pdf` |
| 3 | テキスト番号（四角枠） | 四角枠内の `H350-01` 等 | `H350-01_orig.pdf`, `国語WS-01_orig.pdf` |

いずれにも該当しない場合は、元のファイル名をそのまま出力します。

#### ページ番号

- 各ページのフッターに印刷された連番の数字を検証対象とします
- ページ番号がないページは無視します
- 3ページ以上のファイルのみ検証対象とします（2ページ以下はスキップ）

#### AI（Vision）による補完

テキスト抽出が失敗した場合（スキャン画像のため文字化けや空文字になる場合）は、OpenAI の Vision モデルを使って画像から直接情報を読み取ります。`--no-ai` で無効化できます。
